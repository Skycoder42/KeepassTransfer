Usage

The following table will help you to understand the parameters you can add. If you're not sure how to use them, just keep the default values. They work for most cases.

  • Key-Size

    The Key-Size parameter can be used to change the size of the generated key. The bigger the Key-Size, the more secure it is. In addition to that, the maximum size of the data to encrypt increases. However, if you choose a bigger key, the generation will take much longer, and the QR-Code will get more complex, and thus may require a bigger size.

    The following sizes are supported: [512, 1024, 2048, 4096, 8192]. The default size (if you leave out the parameter) is 2048.

  • Error-Level

    The Error-Level specifies how much error-correction should be used. The higher the error correction, the more damaged the QR-Code can be, without being a problem for readers. More information about the levels can be found at QRStuff.com 🗗.

    The following levels are supported: [L, M, Q, H]. Typically, a QR-Code on the screen is not damaged. To reduce the complexity of the code, the default error correction level is L.

  • QR-Code Size

    The QR-Code Size parameter allows you to adjust the size of the generated QR-Code. it specifies the height and width in pixels. Depending on the Key-Size and Error-Level, as well as the screen-size, you may have to adjust this parameter to get a readable QR-Code.

    All sizes from 100 to 999 are allowed. The default size is 512 pixels.
